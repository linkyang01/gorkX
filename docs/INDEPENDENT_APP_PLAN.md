## 0. 产品北极星（必须对齐 · 用户原话）

```text
功能对齐 Codex（一个完整 coding agent 桌面该有的都要有）
  → 内核用开源 Grok Build 的「源码」：自己编译、自己打包进 gorkX
  → 上游不好用的地方：在 fork/补丁里改掉，再合回我们的引擎构建
  → 用户只装 gorkX，不装 Grok Build / CLI
  → 上游更新可跟，节奏我们掌握；能力跟得上巨人，体验按 Codex 做得更好
```

| 维度 | 定死 |
|------|------|
| **产品** | 你自己的独立桌面 Agent（gorkX） |
| **功能 / UX** | **对齐 Codex** — agent 该有的都要有，更好用 |
| **内核** | **Grok Build 源码** → 自编译 → 打进 App（不是「用户已安装的 grok」） |
| **改引擎** | **允许且鼓励**：fork / patch 掉上游烂点，版本与补丁我们控 |
| **不是** | 只包官方二进制的薄壳；也不是功能残缺的半成品 |

### 0.1 三句话（禁止再理解偏）

1. **功能 = Codex 级**（完整 agent 桌面：项目、任务、权限、审阅、终端、扩展、记忆、工作树、设置…）  
2. **脑子 = Grok 源码内核**（自编译、可改、可跟上游）  
3. **牌子 = gorkX**（用户只认识这一个 App）

### 0.2 引擎策略：源码 · 自编译 · 可改

| 做法 | 要 / 不要 |
|------|-----------|
| 以 [xai-org/grok-build](https://github.com/xai-org/grok-build) **源码树**（submodule / 镜像 / fork）构建引擎 | ✅ 要 |
| CI / `mac-build` **编译**出引擎二进制并打进 `gorkX.app` | ✅ 要 |
| 对上游的 bug、难用点、桌面不适配处打 **补丁或维护 fork 分支** | ✅ 要 |
| 跟上游：merge / cherry-pick，**版本号与节奏我们定** | ✅ 要 |
| 要求用户 `curl \| bash` 安装 Grok CLI | ❌ 不要 |
| 默认 PATH 扫系统 `grok` 当产品路径 | ❌ 不要 |
| 永远零改动、只当官方二进制转发器 | ❌ 不要（与「改掉不好的」矛盾） |

### 0.2.1 上游能力桌面化规则（强制）

每次锁定 Grok Build 新版本，必须逐条审计发布说明，并为每项能力归入以下一类：

| 分类 | gorkX 动作 | 是否可宣传 |
|------|------------|------------|
| ACP/CLI 有稳定接口 | 做成更简洁的桌面入口，并做真实回归 | 通过回归后可以 |
| 仅 TUI 内部实现但对桌面有价值 | 在受控 fork/补丁中补 ACP 接口，再由 UI 接入 | 补丁与回归完成前不可以 |
| 只解决终端场景或不适用于桌面 | 不做重复按钮，记录理由 | 不作为桌面功能 |

例如 0.2.111 所述 macOS Voice Mode 属于第二类：必须复用 Grok Build 的
STT、认证和低内存采集实现，不能用 Web Speech 冒充；在 ACP 扩展完成前，
gorkX 不显示可点击的语音输入入口。

**0.2.111 接入台账（等待上游可复现源码提交）：**

| 上游变化 | gorkX 处理 | 当前结论 |
|------|------|------|
| 图片/视频工具总开关 | 在 App 自管 `config.toml` 提供“媒体工具”设置，并在内核重启后以实际公告的命令为准 | 待内核锁升级后实现与回归 |
| `/session-info` 账户来源 | 用 ACP 的会话信息显示“OAuth / API Key”和对应管理入口；不显示令牌 | 等 0.2.111 的实际 ACP 字段后接入 |
| TUI `/doctor fix` | 仅对与桌面相关、可逆且有明确确认的修复做 UI；绝不静默改 shell 配置 | 现知 `ssh-wrap` 不适用于 App 主路径，保留为终端说明 |
| 会话、权限、MCP、重复调用修复 | 由受控内核更新继承，并加入 ACP 回归场景 | 待源码版本可锁定后验证 |
| macOS Voice Mode 内存优化 | 通过内核 ACP 语音扩展接入麦克风与转写事件 | 已确认当前 ACP 未提供接口，列为内核补丁项 |

**许可：** 上游 Apache-2.0 — 修改与再分发须保留 NOTICE/版权声明（关于页 + 发行物）。

### 0.3 功能对齐 Codex（范围原则）

「一个 agent 有的功能我都要有」= **以 Codex 桌面为功能清单对表**，不是只抄几个按钮：

- 个人：常规、外观、个性化/记忆、快捷键、用量、账户  
- 集成：插件/MCP、浏览器、电脑操控（可分期，但清单上要有位）  
- 编码：钩子、连接、Git、环境、工作树、多任务…  
- 任务：新建 / 归档 / 恢复 / 删除；权限 Default/Auto/Full  
- 工作区：项目、Diff/审阅、终端、过程流  

分期交付可以，**清单不能缩成「能聊天就行」**。引擎侧：Grok 已有能力（tools、plan、memory、worktree、MCP、skills…）须在 gorkX 有入口或等价体验；上游缺的靠 **改源码或产品层补**。

### 0.4 验收对照

| 假独立 / 薄壳（禁止） | 真独立 Agent（验收） |
|----------------------|----------------------|
| 用户先装 Grok Build | **只装 gorkX** |
| 只会调系统 `grok` 二进制 | **源码编译进包**，可带我们的补丁 |
| 功能远少于 Codex | **功能清单对齐 Codex**（可分期，不可永久砍） |
| 上游烂点只能忍 | **能改源码再发** |
| 上游一动就被迫全量跟 | **跟不跟、跟哪版，我们定** |
| 只能 Grok 一家订阅 | **多账户挂起 + 多模型任选**（见 §0.5） |

### 0.5 多订阅 · 多模型（对齐 OpenCode 心智）

**产品要求（用户原话）：**

> 订阅可以同时用 GPT 订阅和 Grok 订阅；还能接三方模型、自己配；参考 OpenCode——几个订阅挂起，可以选多种模型。

| 能力 | 说明 |
|------|------|
| **多账户挂起** | 同时启用 ≥2 套凭证：如 SuperGrok / xAI、OpenAI（API 或兼容订阅通道）、Anthropic、其它 |
| **模型任选** | 任务/会话级选模型；列表按 **提供商分组**（Grok / OpenAI / Claude / 自定义…） |
| **自配三方** | OpenAI-compatible `base_url` + key；Anthropic Messages；本地 Ollama 等 |
| **切换成本低** | 换模型不换「整个 App」；新任务默认上次模型；可中途切换（引擎支持时） |
| **用量分账** | 尽量按账户/提供商显示额度或「当前走哪条线」（能拿到的才显示，不假额度） |

**参考 OpenCode（体验，不抄实现）：**

- 多 provider 并存配置（`provider` / auth 多份）  
- `/models` 或 UI 选择器跨提供商  
- 自定义 provider：`baseURL` + models 表  
- 主模型 + 可选轻量模型（标题/摘要等，后做）

**引擎侧（Grok 源码已有基础，我们包成产品）：**

Grok Build 已支持自定义模型（`[model.*]`）：`base_url`、`api_key`/`env_key`、  
`api_backend` = `chat_completions` | `responses` | `messages`（OpenAI / Anthropic 协议）。  
见上游文档 *Custom Models*。  

| 我们要做的 | 内容 |
|------------|------|
| **设置 → 账户 / 模型** | 挂载多套：Grok 登录、OpenAI Key/兼容、Anthropic、自定义 endpoint |
| **安全** | Key 存 App 钥匙串 / 加密存储，不进 git、不进聊天日志 |
| **写入引擎配置** | 生成/合并 App 自管 home 下的 `config.toml` `[model.*]`，不逼用户手改 TOML |
| **模型选择器** | Codex/OpenCode 式：按提供商折叠，显示「来自哪张订阅」 |
| **源码补强** | 上游缺的多账户体验、订阅 OAuth、额度聚合——**在 fork 里改**，节奏我们控 |

**分期（不挡 I1，但清单锁定）：**

| 阶段 | 交付 |
|------|------|
| **M0** | 设置里配置 1+ 自定义模型（OpenAI 兼容）+ 模型列表出现 + 能切 |
| **M1** | Grok 订阅 + 自定义 OpenAI/兼容 **同时挂起**；选择器分组 |
| **M2** | Anthropic Messages；「添加提供商」向导（名称、base URL、key、模型 id） |
| **M3** | GPT/ChatGPT 类订阅通道（若可合法对接 OAuth/兼容网关则做；否则明确「API Key / 兼容网关」路径） |
| **M4** | 多任务可各用不同模型；可选 small_model 做标题等 |

**诚实边界：**

- 「GPT **订阅**」≠ 一定能直接蹭 ChatGPT Plus 网页登录；优先 **OpenAI API / 兼容中转 / 企业网关**。若官方有桌面可用的订阅 OAuth，再接入。  
- 不装假「已连通 GPT 订阅」；未接通就显示「待配置」。  
- 默认仍可只挂 Grok；多订阅是能力，不是强制。

## 1. 目标架构（在现有 UI 上）

```text
┌─────────────────────────────────────────────────────────┐
│  gorkX.app（现有 Tauri + React UI 继续用）               │
│  侧栏 · 任务 · 审阅 · 过程 · 终端 · 记忆 · 扩展 · 设置   │
└──────────────────────────┬──────────────────────────────┘
                           │ ACP stdio（协议不变）
                           ▼
┌─────────────────────────────────────────────────────────┐
│  内核：Grok Build 二进制                                 │
│  路径优先级（写死）：                                     │
│    1. App 内 Resources/grok（打包必带）                  │
│    2. ~/Library/Application Support/gorkX/runtime/grok │
│    3. 用户在设置中显式指定（高级，非默认）                 │

└──────────────────────────┬──────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────┐
│  数据：gorkX 自管 home（默认）                            │
│  GROK_HOME 或等价环境变量 →                              │
│    ~/Library/Application Support/gorkX/grok-home/        │
│  内含：sessions · auth · memory · config · mcp …         │
│  gorkX SQLite：任务标题、项目绑定、归档标记、UI 偏好      │

└─────────────────────────────────────────────────────────┘
```

**一句话：** UI 还是现在的 gorkX；**进程、目录、项目/会话/记忆归属全部收归 App。**

## 2. 领域模型：项目 · 会话 · 记忆（必须逻辑清晰）

三层对象，**禁止混谈**。

```text
用户
 └── 项目 Project（工作区目录，可选）
      └── 任务/会话 Session（一次 Agent 对话）
 └── 记忆 Memory（跨会话「你是谁 / 你怎么干活」）
      ├── 用户画像 USER（偏好、沟通方式）  ← 越用越像你
      ├── 工作笔记 AGENT（环境、约定、教训）
      └── 会话沉淀 SESSION LOGS（可整理进上面两层）
```

### 2.1 项目 Project

| 字段（概念） | 说明 |
|--------------|------|
| `id` / `path` | 本地文件夹绝对路径（工作区 cwd） |
| `displayName` | 侧栏显示名（默认同文件夹名） |
| `pinned` | 是否置顶 |
| `inApp` | 是否在 gorkX 项目列表（移除列表 ≠ 删磁盘） |

| 用户动作 | 系统行为 |
|----------|----------|
| 添加/打开项目 | 选目录写入 App 列表；**不**自动扫全盘 |
| 新建项目文件夹 | 可在 `~/…/gorkX/projects` 或用户指定处建目录 |
| 重命名 | 改显示名；可选同步改磁盘目录名并 rekey 会话 cwd |
| 从列表移除 | 只移出 UI 列表；**不删**磁盘、**不删**该项目下已归档任务（可配置） |
| 置顶 | 仅排序 |

**规则：**

- 「项目」= **工作区**，不是会话容器 alone；会话挂在项目下或「无项目任务」。  
- 无项目任务：cwd = 用户主目录或设置中的默认目录，`projectKey = __none__`。  
- 项目 **不** 从系统 `~/.grok` 反推；只来自用户在 App 里添加的路径。

### 2.2 会话 / 任务 Session

| 字段（概念） | 说明 |
|--------------|------|
| `id` | App 任务 id（UI 主键） |
| `sessionId` | 内核会话 id（ACP / GROK_HOME 内） |
| `projectKey` | 所属项目 path 或 `__none__` |
| `title` | 首条用户消息锁定，可手改 |
| `archived` | 归档标记 |
| `updatedAt` | 排序 |

| 用户动作 | 系统行为 |
|----------|----------|
| 新建任务 | 内核 `session/new`（cwd=项目）+ 写 App 索引 |
| 打开任务 | 内核 `session/load` + 展示本地快照/流 |
| 归档 | `archived=true`，侧栏主列表隐藏 |
| 从归档恢复 | `archived=false`，回到主列表 |
| 删除 | 删索引 + 内核删除会话（确认）；**不可恢复** |
| 导出 | 导出该会话 Markdown（可选） |

**规则：**

- 侧栏只显示：**本 App 创建且未归档** 的任务。  
- **禁止**主路径「导入内核 / 扫全库灌列表」。  
- 高级「迁移旧 CLI 数据」= 设置里一次性向导，默认关。  
- 标题：首次有效用户消息定稿，禁止每轮自动改名。

### 2.3 记忆 Memory（越用越像你）

#### 2.3.1 产品目标（学 Hermes，不整仓换 Hermes）

参考 [Hermes Persistent Memory](https://hermes-agent.nousresearch.com/docs/user-guide/features/memory)：

| Hermes 做法 | gorkX 采用 |
|-------------|------------|
| **USER** 画像：偏好、沟通风格、忌讳 | **`USER.md`（用户画像）** — 越用越像你 |
| **MEMORY** 笔记：环境、约定、教训 | **`AGENT.md` / 工作笔记** — 越用越懂你的环境 |
| 严格容量，满则合并/删旧，不静默丢 | UI 显示用量；满时引导整理（`/dream` 或面板） |
| 开局注入 system，本局写入盘、下局生效 | 与 Grok 会话启动一致：依赖内核 memory 工具 + 文件 |
| 主动沉淀：纠正、偏好、约定 | 开「自动学习」时：结束会话/flush 触发沉淀 |
| 会话全文检索 ≠ 精炼记忆 | **精炼记忆** vs **任务历史**：两套入口 |

**用户一句话验收：**

> 用两周后，Agent 默认就按我的口味写代码、按我的仓库约定做事，不用每次再说一遍。

#### 2.3.2 存储位置（全部在 App 自管 home 内）

默认根目录：

`~/Library/Application Support/gorkX/grok-home/`

（即独立 `GROK_HOME`，**不是**用户系统 `~/.grok`，除非用户显式迁移。）

| 逻辑层 | 建议路径（在 GROK_HOME 下） | 内容 |
|--------|---------------------------|------|
| 用户画像 | `memory/USER.md` | 称呼、语言、回复长短、技术栈偏好、禁忌 |
| 工作笔记 | `memory/AGENT.md` 或沿用 Grok `MEMORY.md` 全局段 | 环境、工具链、通用习惯 |
| 项目记忆 | `memory/workspaces/<slug>/MEMORY.md` | 该仓库约定、命令、坑 |
| 会话沉淀 | `memory/.../sessions/` | flush 产生的摘要，可经整理并入上两层 |
| 索引 | SQLite FTS（内核已有则复用） | 搜索 |

若官方 Grok 仅认 `~/.grok/memory` 布局：在 **GROK_HOME 内保持相同相对结构**，使内核 memory 功能原生可用。

#### 2.3.3 记什么 / 不记什么（写进产品与提示策略）

**要记（主动或确认后）：**

| 类型 | 写入 | 例 |
|------|------|-----|
| 沟通偏好 | USER | 要简洁、用中文、少客套 |
| 技术偏好 | USER / 项目 | 偏 TypeScript、讨厌无类型 any |
| 纠正 | USER 或 AGENT | 「不要用 sudo 跑 docker」 |
| 项目约定 | 项目 MEMORY | 测试命令、目录结构、PR 规范 |
| 环境事实 | AGENT | 本机工具链、常用路径 |
| 显式「记住」 | 对应层 | 用户点「记一条」或说 remember |

**不要记：**

- 一次性调试路径、临时报错全文  
- 密钥、Token、隐私（检测拦截）  
- 可随时 web 查到的通识  
- 超长日志/代码块  

#### 2.3.4 怎么记（用户可见流程）

| 方式 | 行为 | UI |
|------|------|-----|
| **显式** | 用户「记一条」/ 说记住… | 记忆面板；可预览再确认（Hermes write_approval 精神） |
| **会话结束沉淀** | 非琐碎会话结束或手动「写入记忆」 | 调用内核 `/flush` 等价能力 |
| **整理** | 合并碎条目、提高信息密度 | 「整理记忆」→ `/dream` 或 App 触发 |
| **自动学习（可关）** | 检测到稳定偏好/纠正时提议写入 | 默认建议 **开**，重要写入可要求确认 |
| **开局加载** | 新任务启动时注入 USER + 当前项目 MEMORY | 内核 memory on + 正确 GROK_HOME |
| **遗忘** | 用户删条目或说「忘掉…」 | 面板编辑 / 内核 forget |

#### 2.3.5 记忆与项目 / 会话的边界

| | 项目 | 会话 | 记忆 |
|--|------|------|------|
| 生命周期 | 跟文件夹/列表 | 跟一次对话 | 跨会话长期 |
| 删除项目列表 | 不删磁盘；记忆是否保留可配置 | — | 项目 MEMORY 默认可留 |
| 删除会话 | — | 删对话；**不自动清空** USER/项目 MEMORY | 已沉淀的保留 |
| 归档会话 | — | 隐藏对话 | 不影响记忆 |

## 3. 会话模型（主路径）

### 3.1 每天用

| 动作 | 含义 | UI |
|------|------|-----|
| 新建任务 | 内核 new session + App 索引；**携带记忆注入** | 新建 / 首页发送 |
| 打开任务 | load 本索引 sessionId | 侧栏任务 |
| 归档 | 隐藏，可恢复 | 归档 |
| 恢复归档 | 回主列表 | 已归档 |
| 删除 | 索引 + 内核真删 | 删除 + 确认 |

### 3.2 禁止当主路径

| 去掉或降级 | 原因 |
|------------|------|
| 「导入内核会话」一等入口 | 两套世界，破坏独立与「我的任务」 |
| 自动灌系统 `~/.grok/sessions` | App 变成 CLI 浏览器 |

### 3.3 可选高级（默认关）

- 从旧系统 `~/.grok` **一次性迁移**到 App GROK_HOME  
- 开发用：指定外部 grok 二进制

## 4. 分阶段交付（只做真独立 + 清晰模型）

### 阶段 I0 — 诚实与止血（0.5–1 天）

**出口：** 打开 App 不会再被「导入内核」带偏。

### 阶段 I1 — 运行时独立（核心，3–5 天）

**目标：** 干净机器只装 gorkX 能跑通一轮对话。

| # | 工作 | 验收 |
|---|------|------|
| I1.1 | 打包脚本：把 `grok`（或官方 release 二进制）打进 `Contents/Resources/grok` | 包内存在可执行文件 |
| I1.2 | 启动时解析内核：**仅** Resources → Application Support/runtime → 设置覆盖 | 默认不依赖 PATH |
| I1.3 | 启动 Agent 时设置 **GROK_HOME**（或 Grok 文档规定的等价变量）指向 App 数据目录 | 新会话写入 gorkX 目录，不污染用户全局 CLI 库（默认） |
| I1.4 | 首次启动：若无内核，引导下载/解压到 runtime（或安装包已带则跳过） | 缺内核有明确 UI，不是静默失败 |
| I1.5 | 设置页：显示内核路径、版本、数据目录；「检测更新内核」 | 可核对「到底用的哪份 grok」 |
| I1.6 | Doctor：一键检测内核可执行、版本、GROK_HOME、登录态 | 排障不靠猜 |

**出口（硬验收）：**

```text
新用户 Mac（无 ~/.grok/bin/grok 或故意改名隐藏）
  → 安装 gorkX.app
  → 打开 → 登录
  → 选项目 → 发一条消息 → 有流式回复
  → 关掉重开 → 任务还在，可继续
```

任一失败 = **阶段未完成**。
